# 超群签收 — 系统架构

## 1. 系统定位

粮油批发送货签收数字化系统，核心解决：
- 送货单签收从纸质三联单迁移到 H5 电子签名
- 保留纸质打印能力（部分客户仍需纸质票）
- 与金蝶云星辰 ERP 双向同步
- 通过飞书与莱运 agent 联动

## 3. 数据库设计

### 核心表

| 表 | 说明 |
|----|------|
| driver | 司机信息 |
| customer_config | 客户签收配置（数字/纸质/两者） |
| delivery_order | 送货单主表 |
| delivery_item | 送货单明细 |
| sign_record | 签收记录（签名图片、照片） |
| print_job | 打印任务队列 |
| admin_user | 管理员 |
| audit_log | 审计日志 |

### 金额处理

所有金额以 **分（cents）** 存储为 BigInt，避免浮点精度问题。
- 前端显示时除以 100
- API 传入传出均为分

## 4. API 设计

### Webhook（莱运 → 签收系统）

```
POST /api/webhook/order     — 莱运推送新订单
POST /api/webhook/callback  — 金蝶回调
```

### 签收（H5 公开页面，token 认证）

```
GET  /api/sign/:token       — 获取待签收订单信息
POST /api/sign/:token       — 提交签收（签名+照片）
POST /api/sign/:token/reject — 拒收
```

### 司机端

```
GET  /api/driver/orders     — 今日送货单列表
POST /api/driver/orders/:id/deliver  — 标记已送达
POST /api/driver/orders/:id/sms      — 发送签收短信
```

### 管理后台

```
GET    /api/admin/orders          — 订单列表（分页、筛选）
GET    /api/admin/orders/:id      — 订单详情
POST   /api/admin/orders/:id/print — 手动打印
GET    /api/admin/customers       — 客户配置列表
PUT    /api/admin/customers/:id   — 更新客户配置
GET    /api/admin/stats           — 统计数据
GET    /api/admin/audit-log       — 审计日志
```

## 5. 签收流程

```
莱运创建销售出库单(金蝶)
    ↓
莱运 webhook → 签收系统创建送货单
    ↓
司机查看今日送货单 → 出发送货
    ↓
送达后标记"已送达"
    ↓
┌─── 数字签收 ───┐     ┌─── 纸质签收 ───┐
│ 发送短信给客户  │     │ 打印送货单     │
│ 客户打开H5页面  │     │ 客户签字盖章   │
│ 手写签名+拍照   │     │ 司机拍照上传   │
│ 提交签收       │     │ 标记已签收     │
└───────┬────────┘     └───────┬────────┘
        │                      │
        └──────┬───────────────┘
               ↓
    飞书通知莱运 → 签收完成
               ↓
    金蝶状态同步（可选）
```

## 6. 安全设计

### 签收链接安全

- HMAC-SHA256 生成 token，包含订单ID + 过期时间
- 默认 72 小时有效期
- 一次性签收，签收后 token 失效
- 记录签收 IP 和 User-Agent

### 管理后台安全

- iron-session 会话管理
- bcrypt 密码哈希
- RBAC: admin（全权限）、driver（查看/操作自己的单）、viewer（只读）
- 操作审计日志

### Webhook 安全

- 入站 webhook 使用 HMAC 签名验证
- 请求频率限制

### 存储安全

- OSS 私有 bucket
- 签名图片通过临时签名 URL 访问（15 分钟有效）
